## 🎯 Краткое резюме

### Что происходит при `npm run dev`:

```
1. npm читает package.json → находит "dev": "vite"
2. Запускает команду vite
3. Vite создает HTTP сервер (порт 5173)
4. Vite открывает WebSocket для HMR
5. Вы открываете браузер → http://localhost:5173
6. Браузер загружает файлы (Vite компилирует TS на лету)
7. Вы редактируете файл → Vite видит изменение
8. Vite через WebSocket обновляет браузер
9. Изменения видны мгновенно! ⚡
```

### Почему Vite быстрый:

- ⚡ **esbuild** (Go) - компилирует в 10-100x быстрее
- ⚡ **ESM** - браузер сам загружает модули
- ⚡ **Кеширование** - зависимости собираются один раз
- ⚡ **HMR** - обновляет только измененные модули
- ⚡ **Lazy loading** - загружает только нужное

### Основные свойства Vite:

| Свойство | Описание |
|----------|----------|
| **Dev Server** | HTTP сервер для разработки |
| **HMR** | Обновление без перезагрузки |
| **Build** | Оптимизация для продакшна |
| **TypeScript** | Встроенная поддержка |
| **CSS/SCSS** | Автоматическая обработка |
| **Plugins** | Расширение функциональности |

## 💡 Подробное объяснение (кратко)

### Что такое npm?
**npm** (Node Package Manager) - это менеджер пакетов для JavaScript. Он устанавливает библиотеки и инструменты в ваш проект.

```bash
npm install        # Установить все зависимости
npm run dev        # Запустить скрипт "dev"
npm run build      # Запустить скрипт "build"
```

### Что такое package.json?
Это файл с информацией о проекте:
```json
{
  "name": "ts_basic",           // Имя проекта
  "scripts": {                   // Команды
    "dev": "vite",              // npm run dev запустит vite
    "build": "tsc && vite build"
  },
  "devDependencies": {          // Зависимости для разработки
    "vite": "^6.4.1",
    "typescript": "^5.2.2"
  }
}
```

### Что такое Vite?
**Vite** - современный инструмент для разработки веб-приложений.

**Преимущества:**
- ⚡ Мгновенный старт (0.1-0.3 секунды)
- 🔄 Быстрое обновление (HMR за 50-200ms)
- 📦 Простая настройка
- 🎯 Встроенная поддержка TypeScript, JSX, CSS

**Как работает:**
1. Создает локальный сервер (http://localhost:5173)
2. Компилирует файлы на лету (когда браузер их запрашивает)
3. Отслеживает изменения и обновляет браузер через WebSocket

### Что такое TypeScript?
**TypeScript** - это JavaScript с типами.

```typescript
// JavaScript (без типов)
function add(a, b) {
  return a + b;
}

// TypeScript (с типами)
function add(a: number, b: number): number {
  return a + b;
}
```

**Преимущества:**
- 🐛 Меньше ошибок (проверка на этапе разработки)
- 💡 Лучшая автодополнение в редакторе
- 📚 Самодокументируемый код

### Что такое Dev Server?
**Dev Server** - локальный веб-сервер на вашем компьютере для разработки.

```
Когда вы запускаете npm run dev:
1. Vite создает HTTP сервер на порту 5173
2. Вы открываете http://localhost:5173 в браузере
3. Видите свой проект как на настоящем сайте
4. При изменениях - автоматическое обновление
```

### Что такое HMR (Hot Module Replacement)?
**HMR** - обновление модулей без перезагрузки страницы.

```
Обычная разработка:
Изменение → Ctrl+S → F5 (перезагрузка) → Потеря состояния

С HMR:
Изменение → Ctrl+S → Мгновенное обновление → Состояние сохранено!
```

**Пример:**
Вы заполняете форму, меняете CSS кнопки - форма остается заполненной!

### Что такое ESM (ES Modules)?
**ESM** - стандарт JavaScript для модулей.

```javascript
// Старый способ (CommonJS)
const fs = require('fs');
module.exports = { myFunction };

// Новый способ (ESM)
import fs from 'fs';
export { myFunction };
```

**Почему важно:**
Современные браузеры понимают ESM нативно → не нужен bundler в dev режиме!

### Что такое Build (сборка)?
**Build** - подготовка кода для продакшна (публикации на сайте).

```bash
npm run build
```

**Что происходит:**
1. TypeScript → JavaScript
2. Объединение файлов (bundling)
3. Минимизация (удаление пробелов, короткие имена)
4. Оптимизация (удаление неиспользуемого кода)
5. Результат → папка `dist/`

**Сравнение:**
```
До build (разработка):
- main.ts: 500 строк
- counter.ts: 200 строк
- Всего: ~50 KB

После build (продакшн):
- index-HASH.js: 1.33 KB (минимизирован + gzip)
- Загружается в 37x быстрее!
```

## 🔄 Полная схема работы (простым языком)

```
1. ВЫ ПИШЕТЕ КОД
   counter.ts (TypeScript)
   ↓

2. ИМПОРТИРУЕТЕ В MAIN.TS
   import { setupCounter } from './counter'
   ↓

3. HTML ПОДКЛЮЧАЕТ MAIN.TS
   <script type="module" src="/src/script/main.ts">
   ↓

4. ЗАПУСКАЕТЕ DEV СЕРВЕР
   npm run dev
   ↓

5. VITE СТАРТУЕТ
   - HTTP сервер (5173)
   - WebSocket (HMR)
   ↓

6. ОТКРЫВАЕТЕ БРАУЗЕР
   http://localhost:5173
   ↓

7. БРАУЗЕР ЗАПРАШИВАЕТ ФАЙЛЫ
   GET /src/script/main.ts
   ↓

8. VITE КОМПИЛИРУЕТ
   TypeScript → JavaScript
   ↓

9. БРАУЗЕР ПОЛУЧАЕТ JS
   Выполняет код
   ↓

10. ВЫ РЕДАКТИРУЕТЕ ФАЙЛ
    Меняете counter.ts
    ↓

11. VITE ВИДИТ ИЗМЕНЕНИЕ
    File Watcher срабатывает
    ↓

12. VITE КОМПИЛИРУЕТ СНОВА
    counter.ts → JavaScript
    ↓

13. WEBSOCKET → БРАУЗЕР
    "Обновить counter.ts!"
    ↓

14. БРАУЗЕР ОБНОВЛЯЕТ
    Только измененный модуль
    БЕЗ перезагрузки страницы!
    ↓

15. ✅ ИЗМЕНЕНИЯ ВИДНЫ МГНОВЕННО!
```

## 📊 Сравнение с другими инструментами

### Webpack (старый подход):
```
Запуск → Собирает ВСЕ файлы → Ждем 20-30 секунд → Браузер
Изменение → Пересборка → Ждем 3-5 секунд → Обновление
```

### Vite (новый подход):
```
Запуск → Мгновенно → Браузер (0.1 секунды)
Изменение → HMR → Обновление (50-200ms)
```
