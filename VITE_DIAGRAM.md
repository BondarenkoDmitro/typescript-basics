## ⚡ Скорость Vite (сравнение)

```
┌──────────────────────────────────────────────────────────┐
│               Время старта dev сервера                   │
├──────────────────────────────────────────────────────────┤
│  Webpack:  ████████████████████████  20+ секунд          │
│  Parcel:   ██████████████  10-15 секунд                  │
│  Vite:     ██  0.1-0.3 секунды  ⚡                        │
└──────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────┐
│               Время HMR (обновления модуля)              │
├──────────────────────────────────────────────────────────┤
│  Webpack:  ████████  3-5 секунд                          │
│  Parcel:   ██████  2-3 секунды                           │
│  Vite:     █  50-200ms  ⚡                                │
└──────────────────────────────────────────────────────────┘

Почему Vite быстрее?
• esbuild (Go) вместо Babel (JavaScript) → 10-100x быстрее
• ESM нативный → браузер сам загружает модули
• Не собирает все → обрабатывает только запрошенные файлы
• Кеширование зависимостей → один раз собрал, много использовал
```

## 🔍 Что такое ESM (ES Modules)?

### Старый способ (CommonJS):

```javascript
// Нужен bundler (Webpack)
const counter = require('./counter.js')
module.exports = { setupCounter }
```

### Новый способ (ESM):

```javascript
// Браузер понимает нативно!
import { setupCounter } from './counter.js'
export { setupCounter }
```

### Поддержка браузерами:

```html
<!-- Все современные браузеры понимают type="module" -->
<script type="module" src="/src/main.js"></script>

Поддерживается с:
✅ Chrome 61+ (2017)
✅ Firefox 60+ (2018)
✅ Safari 11+ (2017)
✅ Edge 79+ (2020)
```

**Vite использует ESM в dev режиме!**

## 🎯 Итог в одной схеме

```
ВЫ пишете:
  npm run dev
       ↓
NPM запускает:
  vite
       ↓
VITE:
  1. Запускает HTTP сервер (5173)
  2. Запускает WebSocket (HMR)
  3. Ждет запросов
       ↓
БРАУЗЕР открывает:
  http://localhost:5173
       ↓
БРАУЗЕР запрашивает файлы:
  GET / → index.html
  GET /style.css → CSS из public/
  GET /src/script/main.ts → Компиляция TS→JS
  GET /src/script/counter.ts → Компиляция TS→JS
       ↓
ВЫ редактируете файл
       ↓
VITE видит изменение:
  1. Компилирует файл
  2. WebSocket → Браузер
       ↓
БРАУЗЕР обновляет модуль
  БЕЗ перезагрузки! ⚡
       ↓
✅ Изменения видны МГНОВЕННО!
```

## ⚡ Швидкість Vite (порівняння)

```
┌──────────────────────────────────────────────────────────┐
│               Час старту dev сервера                     │
├──────────────────────────────────────────────────────────┤
│  Webpack:  ████████████████████████  20+ секунд          │
│  Parcel:   ██████████████  10-15 секунд                  │
│  Vite:     ██  0.1-0.3 секунди  ⚡                        │
└──────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────┐
│               Час HMR (оновлення модуля)                 │
├──────────────────────────────────────────────────────────┤
│  Webpack:  ████████  3-5 секунд                          │
│  Parcel:   ██████  2-3 секунди                           │
│  Vite:     █  50-200ms  ⚡                                │
└──────────────────────────────────────────────────────────┘

Чому Vite швидший?
• esbuild (Go) замість Babel (JavaScript) → 10-100x швидше
• ESM нативний → браузер сам завантажує модулі
• Не збирає все → обробляє тільки запитані файли
• Кешування залежностей → один раз зібрав, багато використав
```

## 🔍 Що таке ESM (ES Modules)?

### Старий спосіб (CommonJS):

```javascript
// Потрібен bundler (Webpack)
const counter = require('./counter.js')
module.exports = { setupCounter }
```

### Новий спосіб (ESM):

```javascript
// Браузер розуміє нативно!
import { setupCounter } from './counter.js'
export { setupCounter }
```

### Підтримка браузерами:

```html
<!-- Всі сучасні браузери розуміють type="module" -->
<script type="module" src="/src/main.js"></script>

Підтримується з:
✅ Chrome 61+ (2017)
✅ Firefox 60+ (2018)
✅ Safari 11+ (2017)
✅ Edge 79+ (2020)
```

**Vite використовує ESM в dev режимі!**
